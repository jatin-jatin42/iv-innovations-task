Assignment: ERP System – Purchase Order (PO) Management System
Objective:
You are to build a Purchase Order (PO) Management System as independent micro-service or in
integration with Odoo/NextERP. This requires tight integration between Vendors, Products, and
Orders.
Main Requirements
1. Backend (Python/PostgreSQL)
a. Create a RESTful API using Python (FastAPI or Flask preferred).
b. Database Schema: Design a PostgreSQL schema with at least three related tables:
• Vendors (Name, Contact, Rating)
• Products (Name, SKU, Unit Price, Stock Level)
• PurchaseOrders (Reference No, VendorID, Total Amount, Status)
c. Business Logic: Implement a "Calculate Total" function that automatically applies a
5% tax and updates the Total Amount based on the items added to the PO.
2. Frontend (JS/TS, HTML5, CSS3, Bootstrap)
a. Build a dashboard to view all Purchase Orders.
b. Create a form to "Create New PO" that fetches a list of Vendors and Products via API.
c. Dynamic UI: Use jQuery or Vanilla JS to allow users to add multiple product rows to a
single order before submitting.
d. Authentication using OAuth with JWT using Google or Microsoft identity provider or
any public IDP.
3. Gen AI Integration (The "Smart" Element) - Optional
a. AI Feature: Add a small "Auto-Description" button next to the Product name.
b. Requirement: Use a simple prompt (simulated or using a free API like Gemini or
OpenAI) to generate a professional 2-sentence marketing description for the product
based on its name and category.
